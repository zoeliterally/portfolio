# Freelance Work

## Positioning

I support small IT and engineering teams with internal tools, platform automation, and practical AI-assisted engineering workflows.

The best fit is focused project work where recurring operations, support, or infrastructure tasks can be turned into reliable software, automation, documentation, or reviewable workflows.

## Focus Areas

### Internal Tools & APIs

- Small internal applications, APIs, CLIs, and admin tooling.
- Java/Spring Boot, Python/FastAPI, Flask, Go, REST APIs, OpenAPI, PostgreSQL, MariaDB, and SQLite.
- Integrations around ticketing, documentation, identity, monitoring, backup, domain, and operational systems.
- Authentication and access-aware tooling with Microsoft Entra ID, OAuth2/OIDC, sessions, and RBAC.

### Platform Automation

- Docker Compose, Traefik, service health checks, deployment structure, and operational handover.
- Ansible-based automation for server baselines, agent rollout, recurring maintenance, and reproducible setup.
- GitLab CI/CD patterns for builds, checks, and container delivery.
- Runbooks, maintenance notes, and documentation that make operational work repeatable.

### Monitoring, Backup & MSP-Adjacent Consulting

- Consulting and development around monitoring, alerting, backup-adjacent workflows, restore thinking, and operational documentation.
- Experience with Checkmk Distributed/MSP, Veeam service-provider contexts, repository servers, Wazuh, Linux, and Hetzner Dedicated/Cloud.
- Clear boundary: consulting, development, documentation, and workflow support only. I do not offer managed MSP service operation, customer resale packages, or 24/7 operational responsibility.

### AI-Assisted Engineering

- Practical AI workflows for code exploration, documentation, review, debugging, architecture options, and repository-level implementation support.
- Safe usage patterns for engineering teams: context discipline, no secrets in external tools, human ownership of architecture, validation, security, and production decisions.
- Setup support for prompts, review checklists, documentation workflows, and agentic coding practices.

## Suitable Projects

- Ops automation audits and implementation sprints.
- Small internal tools that replace scattered scripts or manual workflows.
- API integrations between operational systems.
- Docker, Traefik, Ansible, and CI/CD cleanup for small platforms.
- Monitoring and backup workflow reviews, runbooks, and supporting tools.
- AI-assisted engineering workshops or workflow setup for small technical teams.

## Availability

I am available for clearly scoped freelance projects, primarily on weekends and asynchronously. This works best for bounded implementation, consulting, review, documentation, and automation work.

I am not a fit for open-ended staff augmentation, on-call operations, or urgent production responsibility that requires weekday availability.

## Boundaries

- No use of employer resources, internal information, customer data, private URLs, ticket exports, or confidential operational details.
- No customer acquisition, resale, or managed-service packages that conflict with current employment.
- No projects involving direct conflicts of interest with current employer responsibilities.
- No 24/7 service operation unless separately discussed and explicitly agreed, which is generally outside the intended scope.

## Good Starting Points

| Need | Possible format |
| --- | --- |
| Too many manual support or ops tasks | Ops automation audit or implementation sprint |
| Internal scripts need to become maintainable tools | Internal tool or API project |
| Docker/server setup is hard to hand over | Platform review and cleanup |
| Monitoring or backup workflows are unclear | Consulting, runbook, or supporting tool |
| Team wants to use AI without leaking context | AI-assisted engineering workflow setup |
