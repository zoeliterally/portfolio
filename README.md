## Working Style

I like practical systems: small enough to understand, automated enough to repeat, documented enough to operate, and observable enough to debug under pressure. I care about clear interfaces, boring deployments, useful logs, and tools that reduce interruptions for the whole team.

I also care about the people around the systems: mentoring, review, shared ownership, and documentation that makes the next handover easier.
