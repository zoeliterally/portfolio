# Internal Tooling & APIs

## Context

Technical teams often rely on scattered scripts, manual ticket work, inconsistent authentication, and undocumented operational knowledge. I like turning those rough edges into small systems with clear ownership and production behavior.

## Work

- Built SSH access management tooling with a web application, centralized configuration, role-aware access, export flows, and a cross-platform CLI.
- Implemented API/auth foundations for internal tools using Microsoft Entra ID, database-backed sessions, OpenAPI documentation, and event-oriented thinking.
- Built maintenance automation that connects documentation data with ticket workflows and recurring operational tasks.
- Built feedback and review-oriented tooling around support tickets.
- Used Go for lightweight agents and CLIs where small deployment footprint and cross-platform behavior mattered.

## Technologies

```text
Java · Spring Boot · Go · Python · FastAPI · Flask · PostgreSQL
MariaDB · SQLite · Flyway · Alembic · Microsoft Entra ID · OAuth2/OIDC
Docker · Traefik · GitLab CI/CD · REST APIs · OpenAPI
```

## Outcome

- Centralized workflows that had previously lived in individual scripts or local configuration.
- Better auditability and handover for recurring support and infrastructure work.
- Internal tools that match real operations instead of idealized demos.
